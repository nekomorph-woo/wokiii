# 06 — 决策账本（Decision Ledger）

## 背景

管道各阶段文档中有大量 `[DECISION]` 标记的决策，但散落在不同文档中，无法跨文档聚合查看。人类记忆不可靠，需要一个统一的决策视图。

## 决策

- 在**概览 tab** 新增"决策账本" section（不新增独立 tab）
- 扫描所有已解析文档中的 `[DECISION]` 段落
- 聚合为表格：决策内容、来源文档、来源章节、影响范围

## 数据来源

`document-format.md` 已定义 `[DECISION]` 语义标记，dashboard render.js 已将其渲染为黑色边框卡片。只需要在解析时提取并聚合。

每个 `[DECISION]` 提取：

| 字段 | 来源 |
|------|------|
| 内容 | `[DECISION]` 后的文本 |
| 来源文档 | 当前解析的文件名 |
| 来源章节 | 最近的上一个 `#` / `##` 标题 |
| 上下文 | 前后文摘要（用于快速理解） |

## UI 设计

在概览 tab 的 pipeline 进度条下方新增 section：

```
┌─────────────────────────────────────────────────────┐
│ 📋 决策账本                              共 12 条决策 │
├─────────────────────────────────────────────────────┤
│ # │ 决策内容              │ 来源           │ 章节    │
│───│──────────────────────│────────────────│─────────│
│ 1 │ 不做 PR review...     │ _define.md     │ 非目标  │
│ 2 │ review-engine 编排... │ _registry.md   │ 持久决策│
│ 3 │ simplify 独立命令...   │ check.md       │ R1      │
│...│                      │                │         │
└─────────────────────────────────────────────────────┘
```

- 点击行 → 跳转到对应文档的对应章节（switchTab + scrollToSection）
- 搜索过滤框：支持按内容/来源/章节搜索

## 修改文件

### `render.js`

**新增**：
- `extractDecisions()` — 遍历 `state.parsed`，提取所有 `[DECISION]` 段落
- `renderDecisionLedger()` — 生成表格 HTML，插入概览 tab
- 搜索过滤逻辑

**修改**：
- `renderOverview()` 中调用 `renderDecisionLedger()`

### `style.css`
- 决策账本表格样式
- 搜索框样式
- 行 hover 效果

## 验证

1. 概览 tab 显示决策账本 section，包含所有文档的决策
2. 点击决策行 → 切换到对应 tab → 滚动到对应章节
3. 搜索过滤正常工作
4. 无 `[DECISION]` 标记时不显示空表格（整个 section 隐藏）
