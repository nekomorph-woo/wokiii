# 实体级索引构建指南

## 1. 实体提取规范

对每篇文档提取 5 类实体，构建跨文档引用图谱。

### 1.1 类型实体

| 属性 | 说明 | 示例 |
|------|------|------|
| name | 类型名称 | `DeviceProxyContext` |
| kind | 类别 | class / interface / enum / value-object |
| docId | 定义所在文档 | `03a` |
| section | 定义章节 | `3.2` |
| fields | 字段列表（含类型） | `String virtualFunction` |
| methods | 方法签名列表 | `boolean isOnline()` |

**提取规则**：
- 代码块中的类/接口/枚举声明 → 类型实体
- 同名类型出现在不同文档中 → 标记为"定义"与"引用"
- 字段类型引用其他类型 → 生成关系实体

### 1.2 关系实体

| 关系类型 | 含义 | 方向 |
|----------|------|------|
| depends-on | 编译/运行时依赖 | A → B |
| references | 文档中引用/提及 | A → B |
| implements | 接口实现 | 实现类 → 接口 |
| extends | 继承 | 子类 → 父类 |
| composes | 组合（字段持有） | 容器 → 成员 |

**构建规则**：
- 字段类型、方法参数类型 → composes 关系
- import 声明 → depends-on 关系
- 文档中"详见 XX"、"如 YY 所述" → references 关系
- 所有关系构建有向图，检查循环依赖

### 1.3 契约实体

| 属性 | 说明 |
|------|------|
| name | API/接口名称 |
| module | 所属模块 |
| input | 输入参数（名称+类型+约束） |
| output | 输出结构（名称+类型） |
| preconditions | 前置条件 |
| sideEffects | 副作用（如写操作、状态变更） |
| sourceDoc | 定义位置 |

**提取规则**：
- ACL/Service 的公开方法 → 契约实体
- 调用方文档引用的方法签名 → 与定义方文档交叉比对

### 1.4 场景实体

| 属性 | 说明 |
|------|------|
| id | 场景标识（如 S1、S8_SATISFIED） |
| description | 场景描述 |
| trigger | 触发条件 |
| expectedBehavior | 预期行为 |
| coverage | 已设计 / 仅提及 / 未覆盖 |
| sourceDoc | 设计位置 |

**提取规则**：
- 枚举值、状态码 → 场景实体
- 用例描述、异常路径 → 场景实体
- 文档中"如果...则..."的条件分支 → 场景实体

### 1.5 约束实体

| 属性 | 说明 |
|------|------|
| description | 约束内容 |
| scope | 适用范围（全局/模块/特定类） |
| sourceDoc | 声明位置 |
| explicit | 显式声明 / 隐含假设 |

**提取规则**：
- "必须"、"不允许"、"禁止" → 约束实体
- 分层规则、依赖方向规则 → 约束实体
- 文档中未声明但设计中隐含的前提 → 标记为隐含假设

## 2. 文档层级自动检测

基于文件名和内容特征推断文档在层级中的位置：

| 层级 | 文件名模式 | 内容特征 |
|------|-----------|----------|
| PRD/需求 | prd, req, 需求, requirement | 功能列表、用户故事、验收标准 |
| 架构总览 | arch, overview, 架构, overview | 模块划分、架构图、全局约束 |
| 模块设计 | module, design, 子模块 | 类定义、流程设计、接口签名 |
| 辅助文档 | risk, dependency, tracing, error, refactoring | 风险/依赖/日志/迁移说明 |

**检测失败时**：使用 AskUserQuestion 让用户手动指定文档层级。

## 3. 索引输出格式

`00-review-index.md` 结构：

```markdown
# 文档审阅索引

> 生成时间：{timestamp}
> 审阅范围：{doc-dir}

## 文档清单

| 文档ID | 文件名 | 层级 | 标题 | 实体数 |
|--------|--------|------|------|--------|

## 类型索引

| 类型名 | 类别 | 定义位置 | 引用位置 | 字段数 | 方法数 |
|--------|------|----------|----------|--------|--------|

## 关系图谱

（ASCII 有向图）

## 契约索引

| 契约名 | 所属模块 | 定义位置 | 调用方 |
|--------|----------|----------|--------|

## 场景索引

| 场景ID | 描述 | 覆盖状态 | 设计位置 |
|--------|------|----------|----------|

## 约束索引

| 约束描述 | 适用范围 | 声明方式 | 来源 |
|----------|----------|----------|------|

## 统计

- 类型实体：{N} 个
- 关系边：{N} 条
- 契约：{N} 个
- 场景：{N} 个（已设计 {N} / 仅提及 {N} / 未覆盖 {N}）
- 约束：{N} 条（显式 {N} / 隐含 {N}）
```

## 4. 索引验证

构建完成后自查：

- [ ] 每个类型实体都有定义位置（无悬空定义）
- [ ] 关系图谱中无孤立节点（每个引用都有对应定义）
- [ ] 文档间引用双向可追踪
- [ ] 场景覆盖状态已标注
- [ ] 约束的适用范围已标注
