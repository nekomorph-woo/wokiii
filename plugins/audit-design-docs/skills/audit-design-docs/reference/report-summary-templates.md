## 后续步骤

### 待处理项处理

对于报告中未自动修补的项目：

1. **阅读本报告**中的待处理项，理解每个发现的上下文和影响
2. **逐个决策**：使用 `/grill-me` 对歧义点进行拷问决策，或自行做出设计决策
3. **反向修改设计文档**：按决策结论修改对应的设计文档
4. **验证修复**：修复完成后重新运行 `/audit-design-docs` 验证修复效果

### 重跑验证

```
修复完成后，重新运行：
  /audit-design-docs <设计文档目录>

验证项：
  - 上一轮的 🔴 是否已解决
  - 修补是否引入新的不一致
  - 索引是否需要重建（如果有结构性改动）
```
