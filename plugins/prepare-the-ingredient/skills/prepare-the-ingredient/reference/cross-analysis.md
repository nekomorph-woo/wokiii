# 交叉分析与公共产物提取

批量展开后，扫描全部模块设计，消除重复定义。

## 提取判断标准

| 产物类型 | 提取条件 | 提取目标 |
|----------|----------|----------|
| 数据模型 / 类型 | 2+ 模块定义相同结构 | `modules/_shared/models.md` |
| 工具方法 / 辅助函数 | 2+ 模块引用相同功能 | `modules/_shared/utils.md` |
| 常量 / 配置 | 2+ 模块使用相同值 | `modules/_shared/constants.md` |
| 错误类型 / 异常 | 2+ 模块定义相同异常 | `modules/_shared/errors.md` |

## 执行步骤

1. 读取全部模块的 `design.md`，提取接口契约中的类型、常量、异常定义
2. 跨模块比对，识别重复项
3. 将重复项提取到 `_shared/` 对应文件
4. 更新各模块 `design.md`：删除本地定义，添加对 `_shared/` 的引用
5. 更新 `_registry.md` 的依赖图：标注各模块对 `_shared` 的依赖

## 引用格式

模块 design.md 中引用共享产物：

```markdown
## 共享依赖

- 数据模型：[models](../_shared/models.md)
- 错误类型：[errors](../_shared/errors.md)
```

## 不提取的情况

- 仅 1 个模块使用的类型/方法 — 保留在模块内部
- 模块间接口调用的参数类型 — 那是接口契约的一部分，不是共享产物
- 业务逻辑 — 共享产物不含实现，只含类型定义和常量
