# 代码修改计划模板

> 仅在代码发现模式 Step 3.5 触发时读取。

代码修改计划仅在控制台输出，不持久化为文件。

## 输出结构

```markdown
# 代码修改计划

**来源发现**: <一句话描述发现的问题>
**关联设计文档**: <文档路径:章节>（已修补 ✅）
**决策记录**: <grill-decisions 文件路径 或 "内联于下">

---

## 问题

<详细描述代码中存在的问题，基于 /grill-me 决策结论>

## 根因分析

<从代码和设计文档角度分析问题成因>

## 受影响模块

| 模块 | 文件路径 | 变更类型 | 说明 |
|------|---------|---------|------|
| <模块名> | src/.../Foo.java | 新增方法 | 新增 getDeviceListByInfos |
| <模块名> | src/.../Bar.java | 修改方法 | 调整参数签名 |

## TDD 修复计划

### RED-GREEN 循环

**循环 1**:
- **RED**: 编写测试验证 <预期行为>
- **GREEN**: <最小变更使其通过>

**循环 2**:
- **RED**: 编写测试验证 <下一行为>
- **GREEN**: <最小变更使其通过>

**REFACTOR**: <全部测试通过后的清理工作>

## 验收标准

- [ ] <标准 1（来自 /grill-me 决策）>
- [ ] <标准 2>
- [ ] 新增/修改测试全部通过
- [ ] 现有测试不受影响
- [ ] 代码修改后与设计文档描述一致

## 决策依据

| 决策点 | 用户选择 | 理由 |
|--------|---------|------|
| <from grill-me> | <choice> | <rationale> |
```

## 与 /triage-issue 的字段映射

| 代码修改计划章节 | → | triage-issue Issue 章节 |
|-----------------|---|------------------------|
| 问题 | → | ## 问题 |
| 根因分析 | → | ## 根因分析 |
| TDD 修复计划 | → | ## TDD 修复计划 |
| 验收标准 | → | ## 验收标准 |
| 受影响模块 | → | 补充信息 |
| 决策依据 | → | 补充信息 |
