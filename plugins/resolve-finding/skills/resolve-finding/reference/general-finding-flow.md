# 通用发现模式详细流程

> 本文件为 resolve-finding skill 的通用发现模式提供详细指导。
> 仅在模式检测为"通用发现模式"时由 Claude 读取。

## 1. 理解发现

### 1.1 输入类型

通用发现模式的输入可能为：
- 设计文档中的问题位置（如 `03b-device-proxy-workflow.md §3.2`）
- 自由描述的设计级问题（如"设备选择模块的缓存策略未定义"）
- 多个设计文档间的冲突描述

### 1.2 确认理解

使用 AskUserQuestion 确认对发现的理解：

```
📋 我对此发现的理解：

- **位置**: <设计文档路径:章节>
- **问题**: <复述问题>
- **性质**: 歧义 / 冲突 / 遗漏 / 不一致

是否准确？是否需要补充或修正？
```

---

## 2. 上下文收集

### 2.1 读取设计文档

Read 用户指定的设计文档位置，提取相关内容。

若输入为自由描述（无具体位置）：
- 从描述中提取关键词
- Grep 在设计文档目录中搜索相关章节
- 呈现搜索结果让用户确认

### 2.2 搜索关联点

通过 Grep/Glob 搜索受影响的关联点：

**搜索范围确认**（使用 AskUserQuestion）：
- 仅设计文档目录
- 仅代码目录
- 两者都搜

**搜索策略**：

| 优先级 | 搜索内容 | 目的 |
|--------|---------|------|
| 1 | 关键术语/类名/方法名 | 精确定位相关描述 |
| 2 | 同义词/缩写 | 覆盖不同表述方式 |
| 3 | 上下游依赖文档 | 发现级联影响 |

### 2.3 汇总关联点

将所有关联点汇总呈现：

```
📋 关联点汇总：

| # | 位置 | 类型 | 关联度 | 说明 |
|---|------|------|--------|------|
| 1 | 03a §2.3 | 设计文档 | 高 | 直接定义相关模型 |
| 2 | 03c §1.1 | 设计文档 | 中 | 引用该模型进行选择 |
| 3 | src/.../Foo.java:42 | 代码 | 中 | 实现了文档描述的逻辑 |
| 4 | 05 §3.1 | 设计文档 | 低 | 外部依赖说明中提及 |

是否需要补充或排除某些关联点？
```

---

## 3. /grill-me 上下文构建

通用发现模式中，/grill-me 的输入上下文应包含：

### 3.1 必呈内容

1. **发现描述**：用户原始问题 + 我的理解
2. **设计文档现状**：所有关联点的设计文档原文
3. **代码现状**（若搜索了代码目录）：实际实现
4. **矛盾点**：不同文档间的描述差异

### 3.2 追问重点

- 问题的准确边界在哪里？
- 多个文档描述不一致时，以哪个为准？
- 是否需要新增设计文档章节来澄清？
- 修复后的描述对其他关联点有什么影响？

---

## 4. 反向修补

按 /grill-me 决策修改设计文档。通用步骤见 [common-steps.md](common-steps.md#反向修补)。

通用发现模式中，输出目标由用户在输入阶段选择（设计文档 或 控制台）。选择"控制台"时仅输出修改建议，不写文件。

---

## 5. 连锁检查

### 5.1 关联点覆盖检查

在 Step 2 收集的关联点列表中，检查哪些被本次修改覆盖：

```
📋 连锁检查结果：

| # | 关联点 | 是否已覆盖 | 说明 |
|---|--------|-----------|------|
| 1 | 03a §2.3 | ✅ | 直接修改 |
| 2 | 03c §1.1 | ✅ | 引用已自动一致 |
| 3 | src/.../Foo.java:42 | ❌ | 代码实现未变，但设计文档描述已更准确 |
| 4 | 05 §3.1 | ⚠️ | 可能需要同步更新 |

是否处理 #4？
```

### 5.2 未覆盖项处理

对于未覆盖的关联点：
- 设计文档间的引用不一致 → 建议一并修补
- 代码与设计文档的不一致 → 记录为待处理项，不自动触发代码修改（通用发现模式不修改代码）
- 低关联度项 → 列出但不深入处理

### 5.3 连锁深度控制

通用深度控制规则见 [common-steps.md](common-steps.md#连锁深度控制)。

---

## 6. 状态更新

### 6.1 标记已解决

在用户指定的待处理项位置标记已解决状态：

- 若有具体的文档位置 → 在该位置添加 ✅ 已解决标记
- 若为自由描述 → 在控制台输出解决确认

### 6.2 输出解决摘要

```
✅ 发现已解决

- **决策**: <一句话决策结论>
- **修改文件**: <设计文档路径列表>
- **未覆盖关联点**: <列出未处理的关联点，如有>
```
