# 交接文档格式

## 文件命名

```
.wok-handoff/<YYYYMMDD>-<HHMM>-<概要>-handoff.md
```

示例：`.wok-handoff/20260531-1430-auth-jwt-refactor-handoff.md`

## 文档结构

```markdown
# 交接文档

**时间**: <YYYY-MM-DD HH:MM>
**焦点**: <下一轮 session 目标，无则写"接续当前工作">
**来源**: <当前对话的主要任务>

---

## 进行中

- <当前正在做的任务和进度>（如：auth.py 中间件实现，已完成签发，待实现刷新）

## 已完成

- <本轮已完成的工作>

## 待处理

- [ ] <未完成的任务>
- [ ] <未完成的任务>

## 关键决策

- <决策内容>（如：选用 JWT 双 Token 方案，理由：无状态易扩展）

## 阻塞项

- <未解决且影响后续的问题，无则写"无">

## 参考产物

- <已有产物的路径或 URL，不重复内容>

## 建议技能

- `/<skill-name>` — <使用理由>
```

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 时间 | 是 | 生成时间戳 |
| 焦点 | 是 | 下一轮目标 |
| 来源 | 是 | 当前任务概述 |
| 进行中 | 是 | 未完成任务及当前进度 |
| 已完成 | 否 | 本轮已完成的工作 |
| 待处理 | 是 | checkbox 格式的待办 |
| 关键决策 | 否 | 影响后续工作的决策 |
| 阻塞项 | 是 | 无则写"无" |
| 参考产物 | 是 | 路径引用，不重复内容 |
| 建议技能 | 否 | 下轮 agent 应调用的技能 |

## 示例

```markdown
# 交接文档

**时间**: 2026-05-31 14:30
**焦点**: 完成 JWT 刷新 Token 实现，编写集成测试
**来源**: 用户认证系统重构

---

## 进行中

- auth.py 中间件实现（签发已完成，刷新 Token 待实现）
- 集成测试编写（0/3）

## 已完成

- JWT 双 Token 签发逻辑
- Access Token 校验中间件
- 单元测试全部通过（5/5）

## 待处理

- [ ] 实现 Refresh Token 刷新接口
- [ ] 编写 3 个集成测试（正常刷新、过期刷新、无效 Token）
- [ ] 更新 API 文档

## 关键决策

- Access Token 有效期 15 分钟，Refresh Token 有效期 7 天
- Refresh Token 存储在 httpOnly Cookie

## 阻塞项

- 无

## 参考产物

- `.wok-plans/fix-auth-token-expire/_issue.md`
- `.wok-grill/20260531-1000-DECISIONS.md`
- `src/middleware/auth.py`

## 建议技能

- `/wok-implement` — 继续 TDD 实现刷新接口和集成测试
```
