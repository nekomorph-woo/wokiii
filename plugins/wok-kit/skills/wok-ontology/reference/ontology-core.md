# 本体论驱动的软件开发 — 核心原则

## 六大核心概念

| 概念 | 职责 | 判断标准 |
|------|------|----------|
| **Entity** | 系统世界中可识别、追踪、操作的核心存在 | 有独立生命周期、被多处引用、参与关系 |
| **Attribute** | 描述 Entity 自身的静态或半静态信息 | 分三层：身份 / 运行 / 扩展 |
| **Relation** | Entity 之间的连接方式 | 有类型、有方向、能支持影响分析 |
| **Action** | 对象可以发生的领域行为 | 有前置条件、有结果、有状态变化 |
| **Constraint** | 定义什么可以发生、什么不可以 | 能解释失败原因、集中定义 |
| **State** | 对象生命周期的阶段 | 有限且明确、有合法迁移、有终态 |

## 最高指导规则

1. **对象优先于功能** — 先问"系统世界有什么对象"，再问"做什么功能"
2. **关系优先于字段** — 字段描述对象自身，关系描述对象所在世界
3. **行为优先于按钮** — 先定义领域行为，再设计入口
4. **约束集中表达** — 散落的规则 = 五种版本的历史原因
5. **闭环交付版本** — 版本交付能力闭环，不是零散功能
6. **先具体后抽象** — 先用具体领域对象，共性稳定后再抽象基类

## 反模式清单

| 反模式 | 信号 | 修正 |
|--------|------|------|
| Feature 堆叠 | 版本规划全是功能清单 | 按能力闭环拆分版本 |
| 字段膨胀 | 对象字段越来越多但行为关系没变清晰 | Attribute 分层：身份→运行→扩展 |
| 上帝对象 | 一个对象多个生命周期和职责 | 按生命周期和规则边界拆分 |
| 隐式关系 | 关系藏在 id 字段里无语义 | 显式建模 Relation |
| 规则散落 | 前端/后端/数据库各有一套判断 | 集中定义 Constraint，分层执行 |
| 页面驱动建模 | 先设计页面再反推对象 | 先建领域对象再设计页面表达 |
| 过早泛化 | Entity/Node/Item/Resource 命名 | 先具体对象，共性稳定后再抽象 |

## 能力闭环定义

能力闭环 = 一个用户目标从开始到结束，经过对象创建、关系建立、行为执行、状态变化、规则校验、结果产出，能够形成可验证结果的最小流程。

闭环 ≠ 零散零件（做列表、做按钮、做字段、做页面）。
