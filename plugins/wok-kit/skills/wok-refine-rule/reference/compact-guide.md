# 上下文压缩引导

> **目的**：在合适的时机主动建议压缩，比被动等待自动压缩保留更多有用上下文。

---

## 压缩信号

满足以下任一条件时，在回复末尾追加短语 `This is a good point to compact.` 并附带推荐的压缩指令：

- 多阶段计划中当前阶段已完成，即将进入下一阶段
- 一轮较大规模的工作（实现、调试、重构）刚完成
- 上下文中积累了大量工具输出、日志、探索结果，后续不太需要
- 判断现在压缩比等自动压缩能保留更多有用上下文

**Rationale**: Claude Code 的 prompt suggestion 机制会识别该短语，高概率预测 `/compact` 作为下一步建议，用户一键触发。主动压缩比自动压缩（~95% 容量触发）保留更多有用上下文。

---

## 压缩指令生成

发出压缩信号时，同时输出推荐的 `/compact` 指令，用户直接复制使用。

回顾当前对话，提取以下关键信息作为焦点指令：

| 信息 | 来源 |
|------|------|
| 正在进行的任务 | 对话主题、最近修改的文件 |
| 当前状态和进度 | 已完成步骤、当前步骤 |
| 未完成的工作项 | 待实现、待测试、待修复 |
| 关键决策和理由 | 用户选择、方案对比结论 |

输出格式：

```
📎 建议执行：
/compact 重点保留 <从上表提取的具体内容>
```

示例：

```
📎 建议执行：
/compact 重点保留认证模块 JWT 双 Token 方案决策、auth.py 中间件实现进度（已完成签发，待实现刷新）、3 个待修复的测试用例
```

---

## 压缩后注意事项

压缩后以下内容自动恢复：

| 内容 | 恢复方式 |
|------|----------|
| 项目根 CLAUDE.md + 无 scope rules | 从磁盘重新注入 |
| Auto memory (MEMORY.md) | 从磁盘重新注入 |
| 已调用 skill body | 重新注入（上限 5K tokens/skill，总计 25K tokens） |

以下内容**不恢复**：

| 内容 | 说明 |
|------|------|
| Path-scoped rules | 需重新触发加载条件 |
| 子目录嵌套 CLAUDE.md | 需再次在该子目录中读取文件 |
| 未调用的 skill 描述 | 不会被重新注入 |

压缩后恢复工作时，按需重新读取丢失的上下文（如 path-scoped rules、子目录 CLAUDE.md）。
