# UI 原型产出指南

> 使用 hallmark + DESIGN.md 产出各端 UI 原型的操作指南。

## 前置条件

- hallmark skill 已安装
- DESIGN.md 已生成并放入 `.wok-ui-design/design-system/`

## 核心页面清单

启动 Step 3 前，与用户确认需要原型化的核心页面。建议按优先级分批：

| 优先级 | 典型页面 | 说明 |
|--------|---------|------|
| 高 | 主界面框架 | 导航结构 + 内容区布局 |
| 高 | 核心列表页 | 数据浏览 + 搜索 + 筛选 |
| 高 | 核心编辑页 | 富交互表单或编辑器 |
| 中 | 搜索页 | 搜索交互 + 结果列表 |
| 中 | 详情页 | 数据详情展示 |
| 低 | 设置页 | 配置和偏好 |
| 低 | 导入/导出页 | 数据导入导出 |

## hallmark 调用方式

### 基本 Prompt 结构

```
用 <设计风格> 风格，基于项目的 DESIGN.md，做一个 <产品名> <端> 的 <页面名> 原型。
要求：<具体需求描述>
```

### 多端产出策略

对每个核心页面，分别产出三个端的原型：

```
# 桌面端（macOS 窗口框）
用 hallmark 做一个 <产品名> 桌面端的 <页面名> 原型，macOS 窗口框展示，宽度 1200px。
风格参考项目 DESIGN.md。要求：<具体需求>

# 移动端（iPhone 设备框）
用 hallmark 做一个 <产品名> 移动端的 <页面名> 原型，iPhone 15 Pro 设备框展示。
风格参考项目 DESIGN.md。要求：<具体需求>

# Web 端（浏览器窗口）
用 hallmark 做一个 <产品名> Web 端的 <页面名> 原型，浏览器窗口展示。
风格参考项目 DESIGN.md。要求：<具体需求>
```

### 设备框规格

| 端 | 设备框 | 尺寸 |
|----|--------|------|
| 桌面端 | macOS 窗口（红绿灯） | 1200×800px |
| 移动端 | iPhone 15 Pro（灵动岛） | 393×852px |
| Web 端 | 浏览器窗口（URL 栏） | 1440×900px |

## 原型评审标准

每个原型完成后，按以下标准评审：

| 维度 | 评审要点 | 评分 |
|------|---------|------|
| **品牌一致性** | 是否符合 DESIGN.md 的 token 和风格 | 0-10 |
| **功能完整性** | 是否覆盖核心交互 | 0-10 |
| **视觉层级** | 信息层次是否清晰 | 0-10 |
| **跨端一致性** | 三端设计语言是否统一 | 0-10 |
| **Anti-slop** | 是否有 AI slop 痕迹 | 0-10 |

## 产出文件归档

```
.wok-ui-design/prototypes/
├── desktop/
│   └── <page-name>.html
├── mobile/
│   └── <page-name>.html
└── web/
    └── <page-name>.html
```

文件命名使用 kebab-case（如 `tag-browser.html`、`prompt-editor.html`）。

## 注意事项

- hallmark 通过 `.hallmark/log.json` 记忆历史产出，自动避免重复
- 对某次产出不满意时，可要求「重新生成，换一种布局方式」
- 原型是概念验证，不是生产代码。后续需要转为实际框架实现
