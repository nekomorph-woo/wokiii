# 决策记录格式

## 文件命名

```
.wok-grill/<YYYYMMDD>-<HHMM>-DECISIONS.md
```

示例：`.wok-grill/20260321-1430-DECISIONS.md`

## 文件结构

```markdown
# 决策记录

**主题**: <审视的计划/设计名称>
**时间**: <YYYY-MM-DD HH:MM>
**决策点**: <总数>

---

## 1. <决策点标题>

**问题**: <具体问题>
**推荐方案**: <推荐的答案>
**理由**: <推荐理由>
**用户选择**: <用户最终决定>
**状态**: 已共识 / 待定

---

## 2. <决策点标题>

...
```

## 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| 主题 | 是 | 本次审视的对象 |
| 时间 | 是 | 生成时间戳 |
| 决策点 | 是 | 决策点总数 |
| 问题 | 是 | 追问的具体问题 |
| 推荐方案 | 是 | Claude 提供的推荐 |
| 理由 | 是 | 推荐方案的依据 |
| 用户选择 | 是 | 用户确认的决定 |
| 状态 | 是 | 已共识 / 待定 |

## 状态值

- `已共识`: 用户确认采纳
- `待定`: 用户未当场决定，需后续确认

## 示例

```markdown
# 决策记录

**主题**: 用户认证系统重构
**时间**: 2026-03-21 14:30
**决策点**: 4

---

## 1. 认证方式

**问题**: 采用 JWT 还是 Session 认证？
**推荐方案**: JWT
**理由**: 无状态、易扩展、支持跨域
**用户选择**: JWT
**状态**: 已共识

---

## 2. Token 存储位置

**问题**: 前端 Token 存储在 localStorage 还是 httpOnly Cookie？
**推荐方案**: httpOnly Cookie
**理由**: 防 XSS 攻击，安全性更高
**用户选择**: httpOnly Cookie
**状态**: 已共识

---

## 3. 刷新机制

**问题**: Token 过期后如何刷新？
**推荐方案**: 双 Token 机制（Access + Refresh）
**理由**: 平衡安全性与用户体验
**用户选择**: 双 Token 机制
**状态**: 已共识

---

## 4. 多设备登录

**问题**: 是否允许多设备同时登录？
**推荐方案**: 允许，最多 5 设备
**理由**: 用户常见需求，限制数量防止滥用
**用户选择**: 待定，需与产品确认
**状态**: 待定
```
