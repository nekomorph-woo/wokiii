# 发现展示格式

## 综合发现

合并双轨探索结果后，按以下分类展示：

```markdown
## 发现汇总

### 用户核心需求
- Feature A：一句话描述（来源：发散/侦察/两者）

### 体验增强
- Feature B：...

### 技术底座
- Feature C：...

### 差异化
- Feature D：...
```

每个 feature 附带：做什么、为什么有价值、复杂度评估（S/M/L）。

## 路线图阶段

精选 feature 按阶段分组时，每个 phase 的格式：

```markdown
## Phase 1 — <主题>

目标：<一句话描述这个 phase 要达成的端到端能力>
模块预估：<1-3 个>

- Feature A：简要描述
- Feature B：简要描述
```
